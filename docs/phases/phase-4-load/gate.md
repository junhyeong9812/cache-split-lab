# Phase 4 — 판정 (gate)

> 입력: spec.md P1~P4 · result.md.

## P1 — 5000↑ 에서 B p95 < C p95, 6000 에서 C 만 드롭>1%: **빗나감 (부분 성립)**

- B p95 < C p95: 5500 (47.9 < 213.1) ✓ · 6000 (156.7 < 293.1) ✓ · **5000 은 역 (112.3 > 109.0)** ✗
- 6000 드롭: C 0.00% (B 0.01%) — **둘 다 안 떨어졌다** ✗

**왜 빗나갔나.** 두 가지를 잘못 봤다:

1. **5000 은 "양쪽 다 아픈데 다른 축으로 아픈" 지점이다.** B 의 p95=112 는 미스 꼬리
   (1,033 미스 × 5ms origin + 스로틀 대기), C 의 p95=109 는 핫 노드 대기다. p95 하나로
   순서를 매기면 동률 근처에서 뒤집힌다. 반면 **p50 은 두 arm 을 선명하게 가른다**:
   B 0.378 vs C 16.0 — **42배**. C 는 요청의 절반 이상이 핫 노드에서 스로틀 큐를 겪는다.
2. **드롭은 재현이 불안정한 신호였다.** 30s 창(Phase 0)에선 C 6000 이 드롭 1.84%,
   60s 창에선 0%. 원인 후보는 여럿이고(창 길이·VU 상한 도달 타이밍·계단 진입 시점의
   큐 상태·생성기 상태) **이 데이터로는 분리하지 못한다 — 반복 측정 없이 창 길이를
   원인으로 지목하지 않는다.** 확실한 것은 결과뿐이다: 같은 명목 포화가 드롭 1.84% 로도
   0% 로도 나왔다. constant-arrival-rate 의 드롭은 "가용 VU 소진"이라는 이산 사건이라
   경계 근처에서 이렇게 흔들리고, 지연 분포는 같은 두 런에서 일관되게 나빴다(p50 159→95ms).
   드롭을 판정 기준으로 등록한 것이 실수라는 교훈(틀린 것 #11)은 이 재현 불안정성만으로
   성립한다.

**지연 축으로 다시 읽으면 역전은 전 지점에서 성립한다**: p50 기준 C/B = 42배(5000) ·
104배(5500) · 2.2배(6000 — B 도 43ms 로 아프기 시작). **"고 skew 에선 키 해시가 먼저,
더 깊게 무너진다"는 서사는 데이터가 지지한다** — 다만 그 증거는 사전 등록한 부등식이
아니라 p50 에 있었다. 등록 기준으로는 빗나감으로 기록한다.

## P2 — hit C ≥ A > B (A−B ≤ 1%p), arm 별 ±0.5%p 평평: **빗나감 (부분 성립)**

- 순위: 4000·4500 에선 C ≥ A > B ✓. 5000↑ 에선 C 가 A 보다 **0.01~0.02%p** 낮다 ✗ (문자 그대로).
  차이가 측정 노이즈 수준이라 실질 순위는 C ≈ A > B — 그러나 "≥" 로 등록했으니 빗나감.
- A−B ≤ 1%p: 전 지점 ✓ (최대 0.50%p — 4500 지점 0.9992−0.9942. s=1.5 에서 RR 중복의
  히트율 세금은 거의 소멸).
- 평평 ±0.5%p: ✗ — 첫 지점(4000)이 전 arm 에서 0.4~1.0%p 눌림. 워밍업(SEED=1)이 측정
  시드(42)의 꼬리 키를 못 데워 첫 지점이 수렴 잔재를 흡수했다. 측정 절차 결함이지
  "히트율이 RPS 의 함수"라는 증거가 아니다 — 4500↑ 만 보면 A·C 는 ±0.01%p, B 는
  0.9942→0.9976 (폭 0.34%p = ±0.17%p)로 등록 밴드(±0.5%p) 안 ✓ (B 는 잔여 수렴이
  4500 이후로도 이어진 흔적 — "±0.1%p 평평"은 아니다).

## P3 — 분포 상수 (B 0.50/0.50, C 0.63/0.37, ±0.01): **일치**

B: 0.499~0.501. C: 0.629/0.371 이 전 지점 소수 셋째 자리까지 동일. **쏠림은 부하와 무관,
분포×분할의 상수다** — Phase 0 의 0.628 과도 일치. 예측이 온전히 맞은 유일한 항목.

## P4 — A 는 6000 까지 드롭<0.5%·p95≤50ms: **일치**

드롭 0%, p95 최대 24.5ms. **관측**: 총 CPU 가 같아도 한 덩어리(0.5×1)가 두 조각(0.25×2)
보다 잘 버텼다. **원인은 이 데이터로 분리되지 않는다** — A 와 B/C 는 쿼터 조각 크기
외에도 라우팅(1홉 균등)·노드당 캐시 크기(10k vs 5k)·미스율·부하 균형이 함께 다르다.
"조각이 작을수록 CFS 스로틀 큐가 일찍 선다"는 가설로 남긴다(검증하려면 같은 arm 에서
쿼터만 바꾼 대조가 필요).

## 종합 — 우열 지도 (s=1.5, 캐시 총량·CPU 총량 고정)

> ⚠️ **탐색적 해석이다 — 사전 등록 판정이 아니다.** 등록 지표(P1)는 p95 하나였고 그걸로는
> 5000 에서 B(112)·C(109)가 사실상 동률(C 근소 우위)이다. 아래 표의 100% 행은 **p50 을
> 사후에 끌어와** 읽은 것 — p50 선택의 근거(요청 절반의 경험 vs 꼬리)는 본문 P1 절에
> 있지만, 지점마다 유리한 지표를 고른 지도라는 비판이 성립한다. 확정하려면 다음 스윕에서
> **복합 판정 규칙(p50·p95·드롭의 우선순위)을 사전 등록**하고 재측정해야 한다.

| 부하 | 승자 (탐색적) | 근거 |
|------|------|------|
| ≤80% (≤4000) | 셋 동률 | p50 0.36~0.37ms, 히트율 차 ≤0.5%p |
| 90% (4500) | A ≈ C > B | B 만 p95 34ms (미스 꼬리) |
| 100% (5000) | A 우위. B vs C 는 축이 갈림 | 등록 지표 p95 론 동률(112 vs 109), 사후 p50 론 B 42배 우위(0.38 vs 16ms) |
| 110~120% | A > B ≫ C | p50·p95 모두 B < C (5500: 0.37/47.9 vs 38.8/213 · 6000: 43/157 vs 95/293) |

**한 줄 결론: s=0 의 정답(키 해시)이 s=1.5 의 오답이 된다.** 히트율 함정(중복)은 사라지고
부하 함정(쏠림)이 나타난다 — 같은 "복제+분할" 결정이 접근 분포에 따라 반대로 채점된다.
그리고 A(스케일업)는 양쪽 함정 모두에 면역이다 — 단일 장애점이라는 별개의 대가를 치를 뿐.
