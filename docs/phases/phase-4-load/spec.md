# Phase 4 — 부하 구간별 트레이드오프 (s=1.5, 80~120%)

> **사전 등록 문서. 이 파일은 Phase 4 본 런을 돌리기 전에 작성됐다.**
> 작성 시각: 2026-07-19 오전, Phase 0 gate 직후 (Phase 0 실측이 예측의 입력 — 사전 등록
> 대상은 아래 P1~P4 뿐이다). 커밋 승인 전이면 Phase 1·0 과 같은 "약한 증거" 각주.
> 마스터: `../../EXPERIMENTS.md` §2 Phase 4 · 기준선: `../phase-0-saturation/gate.md`.

## 1. 질문

포화점의 80~120% 구간에서 **세 배치의 우열 지도**는 어떻게 생겼나. 히트율 축(Phase 1~3)과
용량 축(Phase 0)을 한 표에서 만나게 한다 — "평상시엔 안 보이다가 부하가 차오르면
청구되는 비용"을 지점별로 보인다.

## 2. 방법 (사전 고정)

- 그리드: **4000 / 4500 / 5000 / 5500 / 6000** = A 무릎(5000)의 80/90/100/110/120%.
- 각 지점 **60s** (Phase 0 계단 30s 의 2배 — 지점당 표본 24만~36만).
- 나머지 전부 Phase 0 run2 와 동일: s=1.5 · N=40,000 · CPU 0.5/0.25×2 · 캐시 10,000/5,000×2
  · 스레드 200/100×2 · SEED=42 · k6 런 분리 · `-m 6g` 캡 · 지점 간 카운터 스냅샷 ·
  arm 별 프리웜+워밍업(2,000 RPS·60s) 후 진입.
- 무릎 재판정은 하지 않는다 — Phase 0 소관. 여기선 지점별 p50/p95/드롭/실패/히트/분포를 지도로.

## 3. 사전 등록된 예측

**P1 (역전의 지도 — 핵심)**: 5000 이상의 모든 지점에서 **B 의 p95 < C 의 p95** 이고,
6000 에서 **C 만 드롭 > 1%** (B 는 < 1%). 판정: 두 명제 모두 성립 = 일치.

**P2 (히트율 순위·불변)**: 전 지점에서 hit **C ≥ A > B**, 차이는 A−B ≤ 1%p.
RPS 가 변해도 각 arm 의 hit 는 ±0.5%p 안에서 평평(히트율은 RPS 무관 — DECISIONS §15).

**P3 (부하 분포 상수)**: 전 지점에서 B = 0.50/0.50 (±0.01), C = **0.63/0.37 (±0.01)** —
쏠림은 RPS 의 함수가 아니라 분포×분할의 함수다.

**P4 (A 여유)**: A 는 6000 까지 드롭 < 0.5%, p95 ≤ 50ms — 기준선 120% 에서도 A 는
안 무너진다(무릎 5000 은 p95 10배 기준의 조기 경보였지 붕괴점이 아니다).

**판정 기준**: 각 P 는 명시된 부등식/밴드 그대로. 4개 중 P1 이 이 페이즈의 존재 이유 —
P1 이 깨지면 "역전" 서사 전체를 재검토한다.
