## 특기 사항 (해석은 gate.md)

1. **저부하 p50 = 0.42~0.43ms** — A(0.34~0.36) 대비 +0.07~0.09ms (~1.2배). 같은 호스트
   도커 브리지의 redis 왕복+Lua 는 예측(0.5~1.5ms)보다 훨씬 쌌다.
2. **무릎 너머의 붕괴가 4 arm 중 유일하게 goodput 역행형**이다. A/B/C 는 과부하에서도
   달성이 한계치(6.9k/6.9k/5.9k)에 고정되는데, D 는 목표를 올릴수록 달성이 **떨어진다**
   (3.5k→1.8k). 실패 다수가 ~5s 에서 절단되는 패턴(p95 ≈ 5000ms)과 동반 — 원인 계층
   (k6 gracefulStop 5s 의 스텝말 절단 vs nginx/앱 큐 타임아웃) 미분리, 미해결로 기록.
3. 정상 구간 노드 분포 0.50/0.50 정확, 붕괴 구간에서만 0.526/0.474 로 드리프트
   (과부하 아티팩트 — 실패·드롭이 노드 비대칭으로 발생).
4. 히트율은 계단 진행에 따라 0.9979 → 1.0 까지 수렴 (공유 캐시 10k 가 s=1.5 꼬리까지
   거의 다 담음 — A 와 동일 수준, P1 입력).
