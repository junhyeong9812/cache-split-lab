## LRU = Least Recently Used, 
"가장 오래 안 쓴 걸 버린다." 캐시가 꽉 찾는데 새걸 넣어야 하면 뭔가를 버려야 한다.
뭘 버릴까? LRU의 답은 제일 오랫동안 안건드린 것, 최근에 또 쓸 가능성이 높다.
```java

요청:  A    B      C        A          D
캐시: [A]  [B A]  [C B A]  [A C B]    [D A C]
                            ↑          ↑
                       A를 다시 씀    꽉 참 → 제일 오래 안 쓴 B를 버림
                       → 맨 앞으로
```

Zipf = "소수가 대부분을 먹는" 쏠림 분포,
1등 단어가 2등의 2배, 3등의 3배… 즉 빈도 ∝ 1/순위. 상품 조회, 도시 인구 같은 수치들이 그렇게 보인다.
```java

┌─────┬────────────────────────────┬─────────────────────────┐
│  s  │            성격            │ 상위 10개가 먹는 트래픽 │
├─────┼────────────────────────────┼─────────────────────────┤
│ 0   │ 완전 평평 (다 똑같이 인기) │ 1%                      │
├─────┼────────────────────────────┼─────────────────────────┤
│ 1   │ 고전적 Zipf                │ 약 39%                  │
├─────┼────────────────────────────┼─────────────────────────┤
│ 2+  │ 극단적 쏠림 (플래시 세일)  │ 대부분                  │
└─────┴────────────────────────────┴─────────────────────────┘

이 s가 우리 실험의 핵심 축입니다. 왜냐면 s가 낮으면(평평) 캐시를 쪼갠 손해가 크고, s가 높으면(쏠림) 뜨거운 몇 개는 어차피 작은 캐시에도 다 들어가서 쪼개도 손해가 없거든요. "중복이 낭비냐 기능이냐"가 이 숫자 하나로 갈립니다.
```
